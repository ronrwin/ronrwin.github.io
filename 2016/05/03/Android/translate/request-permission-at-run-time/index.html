权限篇（3）----运行时请求权限

发表于 2016-05-03   |   分类于 Android官网翻译   |     |   阅读次数

本文原文链接：http://developer.android.com/training/permissions/requesting.html

本课将教你：

检查权限
请求权限

从Android 6.0（API 23）开始，用户可以在应用运行时授予权限，不一定在安装时授予。这种方法简化了应用程序的安装过程，用户可以不用在安装或升级时授予权限。这也是的用户对应用功能有更多的控制权。举个例子，用户可以给一个相机应用授予相机权限但是不授予访问文件路径的权限；用户也可以在运行时在应用设置界面撤销权限。

系统权限被分为两个类别，普通或危险。

  • 普通权限不会威胁用户隐私。如果在配置文件中声明了一连串的普通权限，系统会自动全部授予这些权限。
  • 危险权限允许应用访问用户的保密数据。如果声明了危险权限，则需要用户去授予权限。

更多资料请参照权限列表。

在Android的所有版本中，你的应用需要在Manifest中声明所需要的普通或危险权限，如声明权限所说。然而，根据不同的系统版本和应用程序的target SDK不同会有不同的影响：

  • 在ANdroid 5.1以下（target SDK 小于或等于 22）：如果应用声明了危险权限，用户必须在安装时授予权限，否则系统不会安装该应用。
  • 在Android 6.0以上（target SDK 大于或等于 23）：当应用运行时，用户必须逐个授予危险权限。用户可以授予或拒绝权限，应用可以在没获取权限的情况下继续运行。

提示：从Android 6.0开始（API 23）用户可以在运行时撤销权限。请确保你的应用在权限缺失下还可以合理运行。

本课将描述如何使用Support Library检查和请求权限。在Android 6.0，系统也会提供相似的方法。然而，使用Support Library会更简单，因为应用不需要检测当前正在使用哪个Android版本。

检查权限

如果应用需要一项危险权限，在运行需要权限的程序时，必须时时检查是否已经获得授权。用户可能在不确定的时候撤销权限，所以即使应用昨天获取了相机权限，也不能保证今天这个权限还被授予。

请求权限

如果应用需要一项配置文件中申声明的危险权限，它必须请求用户授予该权限。Android提供几种方法去请求权限。调用这些方法会弹出基本对话框，样式由系统定义。

解释为什么应用需要权限

在某些情况下，你需要让用户理解为什么应用需要这些权限。例如，如果用户启动一个摄影应用，用户对于请求相机权限不会感到惊讶，但是用户会迷惑为什么应用需要获取地理位置和联系人的权限。当你请求一项权限时，你需要让用户知道原因。记住，你不会想要用解释来强迫用户；如果你使用太多的解释，用户有可能会删除应用。

一种情形是需要向用户解释的。如果用户不断尝试使用某项功能却不断的拒绝授予权限，那么很可能用户并不知道该功能为何需要该权限，此时就需要向用户解释该权限的必要性。

为了更方便发现什么时候用户需要权限解释，Android提供了方法shouldShowRequestPermissionRationale()。返回True说明应用已经请求过权限但是用户拒绝了。

提示：如果用户拒绝权限请求并且勾选不再询问，这个方法会返回false。如果机器不允许应用拥有权限，该方法也会返回false。

请求所需的权限

如果应用没有获取其所需的权限，应用应调用requestPermissions()请求权限。应用传递一个请求码指代所请求的权限。该方法是异步的：它会立即返回，然后当用户响应了系统弹框之后，系统会以请求码为标记回调处理结果。

处理权限请求结果

系统对话框描述应用所需要的权限组，不会列出具体的权限。例如，当请求READ_CONTACTS权限时，系统对话框只会说明该应用需要连接设备通讯录。用户只需要授予所需要的权限组权限，而不需要多次手动授予同一个权限组下的其他权限。

提示：你仍然需要对每一个权限作请求操作，即使你已经对相同权限组的其他权限授权。因为系统对于权限与权限组的分类在未来是有可能变动的。

例如：配置文件中需要READ_CONTACTS和WRITE_CONTACTS权限，如果已经对READ_CONTACTS授权，则当请求WRITE_CONTACTS时，系统会自动授权。

如果用户拒绝一项权限请求，则需要作相应的处理。比如展示一个对话框告诉用户为什么需要此项权限才能执行用户期望的操作。

当系统询问用户授予权限时，用户可能选择不再提示，这种情况下，任何时候应用调用requestPermissions()请求权限时，系统将自动禁止授权。系统调用onRequestPermissionsResult()回调并传回PERMISSION_DENIED。这意味着，当应用调用requestPermissions()时，用户不会得到相应的交互反馈。
